# Mise à jour du README

Analyse les changements récents du projet et mets à jour le README.md pour qu'il soit cohérent.

## Étapes

1. **Analyse la structure actuelle du projet** via `XcodeLS` ou `Glob`
2. **Lis le README.md actuel**
3. **Compare** la structure documentée avec la structure réelle
4. **Identifie les incohérences** :
   - Dossiers/fichiers renommés
   - Nouvelles fonctionnalités
   - Architecture modifiée
   - Noms de services/clients changés
5. **Mets à jour le README** avec les corrections nécessaires

## Points à vérifier

- Structure du projet (arborescence)
- Noms des services (ExportService, pas TokenService)
- Noms des dossiers (Components, pas Views)
- Fonctionnalités listées
- Filtres d'export disponibles
- Stack technique

## Format

Montre-moi les changements que tu vas faire avant de les appliquer.
